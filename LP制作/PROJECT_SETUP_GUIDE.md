## 🎯 プロジェクト1: 吉田商会

### ステップ1: プロジェクト作成
1. Claude.ai にアクセス
2. 左サイドバーの **「Projects」** をクリック
3. **「+ New Project」** または **「Create Project」** をクリック

### ステップ2: 基本情報入力
- **プロジェクト名**: `吉田商会 - 廃油回収見積システム`
- **説明**（任意）: `廃油回収見積の自動生成システム。完成・運用中。`

### ステップ3: カスタム指示を設定
「Custom Instructions」欄に以下をコピー&ペースト：

```
【プロジェクト概要】
吉田商会向けの廃油回収見積自動生成システムの開発・保守プロジェクト。
完成・運用中のシステムの保守、改善、機能追加を担当。

【技術スタック】
- フロントエンド: HTML5, CSS3, JavaScript (ES6+)
- バックエンド: Google Apps Script
- データベース: Google Sheets
- AI: Anthropic API (Claude)
- ホスティング: Lolipop (d-o.pya.jp/test777/)

【主要機能】
1. AI見積生成
2. 価格計算エンジン
3. 予約カレンダー連携
4. チャットボット
5. メール通知
6. 顧客管理

【開発方針】
- 既存システムの安定運用を最優先
- 変更は慎重に、十分なテストを実施
- セキュリティを重視（APIキー管理、データ保護）
- Google Apps Scriptの実行時間制限に注意
- ユーザビリティの継続的改善

【コミュニケーション】
- 日本語で対応
- 技術的な詳細も平易に説明
- 実装前に必ず確認を取る
- コード変更時は変更箇所を明確に説明
```

### ステップ4: ファイルをアップロード
1. **「Add files」** または **「Upload」** をクリック
2. 以下のファイルをアップロード：
   - `吉田商会/README.md`
   - `吉田商会/CUSTOM_INSTRUCTIONS.md`
   - （既存のHTMLファイルやGASコードがあればそれも）

### ステップ5: 保存
- **「Create Project」** または **「Save」** をクリック

```
【プロジェクト概要】
リッキーモータース（仙台）向けの自動車修理見積自動生成システムの開発・保守プロジェクト。
AI画像解析による損傷診断機能を搭載した完成・運用中のシステムの保守を担当。

【技術スタック】
- フロントエンド: HTML5, CSS3, JavaScript (ES6+)
- バックエンド: Google Apps Script
- データベース: Google Sheets
- AI: Anthropic API (Claude) - 画像解析・損傷診断
- ホスティング: Lolipop (d-o.pya.jp/test777/)

【主要機能】
1. 損傷診断AI（写真解析）

3. 重量税自動調整

【開発方針】

- 画像処理の最適化（サイズ・速度）

【コミュニケーション】

- 実装前に必ず確認を取る
- 画像処理の負荷に注意
```

## 🎯 プロジェクト3: 美容理容システム

### ステップ1: プロジェクト作成
1. 左サイドバーの **「Projects」** をクリック
2. **「+ New Project」** をクリック

### ステップ2: 基本情報入力
- **プロジェクト名**: `美容理容 - AIカウンセリングシステム`
- **説明**（任意）: `理容美容業界向けAIカウンセリング&スタイル提案システム。デモ版完成、開発中。`

### ステップ3: カスタム指示を設定
```
【プロジェクト概要】
理容美容業界向けのAIカウンセリング&スタイル提案システムの開発プロジェクト。
現在デモ版が完成し、営業開始可能な状態。今後の本格実装と機能拡張を担当。

【技術スタック】
- フロントエンド: HTML5, CSS3 (Tailwind風), JavaScript (ES6+)
- バックエンド（予定）: Google Apps Script
- AI: 
  - Anthropic API (Claude) - 顔型分析、スタイル提案、テキスト生成
  - Stability AI API - 画像生成（実装予定）
- ホスティング: Lolipop (d-o.pya.jp/test777/)

【現在の状態】
Phase 1（完成）:
1. AIカウンセリングシステム（顔型診断）
2. テキスト→スタイル提案システム
3. 営業用完全版（ガイド・料金表）

Phase 2-3（予定）:
1. 画像生成機能（Stability AI連携）
2. 予約管理システム
3. LINE連携

【開発方針】
- 段階的リリース: デモ版 → 画像生成版 → 予約連携版
- 顧客フィードバック重視
- コスト意識（Phase 1は追加コストなし）
- セキュリティ重視（APIキー管理）

【現在の優先事項】
1. 営業準備（お問い合わせ情報更新、FTPアップロード）
2. 先方への提示と反応確認
3. フィードバックに基づく改善
4. Phase 2への移行判断

【コミュニケーション】
- 日本語で対応
- 営業的な視点も含めて提案
- 実装前に必ず料金・工数を確認
- 段階的な開発を推奨

【料金プラン】
スタンダード: 初期80,000円 + 月額5,000円
プロフェッショナル: 初期150,000円 + 月額8,000円
オプション: 画像生成+30,000円、予約連携+50,000円、LINE連携+40,000円
```

## 🔄 今後の運用

### プロジェクトの使い方
1. **新しい開発・質問がある場合:**
   - 該当するプロジェクトを開く
   - 新しいチャットを開始
   - プロジェクトの文脈が自動的に読み込まれる

2. **ファイルの追加・更新:**
   - プロジェクト設定から「Add files」
   - 新しいファイルをアップロード

3. **カスタム指示の更新:**
   - プロジェクト設定から編集
   - 開発方針が変わった時に更新

### バックアップ
- 定期的に `project-backup-master.md` を更新
- 重要な変更があったらREADME.mdも更新
- VSCodeにローカル保存

---

## 💡 ヒント

### プロジェクトを効果的に使うコツ
- **1プロジェクト = 1クライアント/システム** で管理すると整理しやすい
- **関連ファイルはすべてアップロード** しておくと、いつでも参照できる
- **カスタム指示は簡潔に** 重要なポイントだけ記載
- **定期的にファイルを更新** 最新の状態を保つ

---

## 🆘 トラブルシューティング

### プロジェクトが作成できない
- 無料プランの場合、プロジェクト数に制限がある可能性
- 不要なプロジェクトを削除するか、プランをアップグレード

### ファイルがアップロードできない
- ファイルサイズ制限を確認
- ファイル形式が対応しているか確認

### カスタム指示が反映されない
- プロジェクトを一度閉じて開き直す
- 新しいチャットを開始する
